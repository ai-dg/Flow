# Commit Policy

## Format

Follow Conventional Commits:

```
<type>: <short description>

<optional body: what changed and why, list subtasks if milestone commit>
```

## Types

- `feat`: new feature
- `fix`: bug fix
- `refactor`: restructuring without behavior change
- `docs`: documentation only
- `chore`: tooling, deps, config
- `test`: tests
- `style`: formatting only

## Rules

- Commit coherent milestones, not half-finished work
- Milestone commits must list completed subtasks in the body
- No vague messages (`fix stuff`, `update`, `wip`)
- **Ask before committing — agent must not commit autonomously**
- **No co-author lines** — agents must not add themselves as co-authors (`Co-authored-by:`, `Generated by:`, or any equivalent)

## Examples

**Good:**
```
feat: placeholder substitution before Gemini call

- Added sensitive field list to persona.ts
- Replaced phone, address, email with placeholders before prompt construction
- Injected real values into LaTeX output post-generation
```

**Bad:**
```
fix stuff
update gemini
wip: trying to fix latex
Co-authored-by: Claude <claude@anthropic.com>
```
